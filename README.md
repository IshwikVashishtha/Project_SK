# DeepAgent — SK v3.0

A hierarchical multi-agent AI system with a single orchestrator, specialist sub-agents each with their own summarisation middleware, file/folder handling, MCP integrations, and three interfaces: CLI, Telegram, and WhatsApp.

---

## Architecture

```
User (Telegram / CLI / WhatsApp)
         │
         ▼
    Orchestrator
    (5-stage intent classifier)
         │
   ┌─────┼──────┬──────┬──────┬──────┐
   ▼     ▼      ▼      ▼      ▼      ▼
Research Media File/ System Conv. MCP
Agent   Agent  Folder Agent  Agent  Agent
               Agent              (ReAct loop
                                   all MCP tools)
```

Every agent has its own `SummarizationMiddleware` — a rolling buffer of recent turns plus a compressed running summary — so no agent's context ever grows unbounded.

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Set up environment

```bash
cp env.template.txt .env
# Edit .env — set OLLAMA_BASE_URL, API keys, MCP toggles
```

### 3. Pull Ollama models

```bash
ollama pull gemma2:2b
ollama pull phi3
```

### 4. Run

```bash
python main.py cli          # terminal
python main.py telegram     # Telegram bot
python main.py whatsapp     # WhatsApp webhook
```

## Changing Which LLM Each Agent Uses

Edit `config/settings.py` or set the matching env vars in `.env`:

```python
AGENT_LLM_CONFIG = {
    "orchestrator": {"provider": "ollama",     "model": "gemma2:2b"},
    "research":     {"provider": "groq",       "model": "llama3-8b-8192"},
    "media":        {"provider": "ollama",     "model": "phi3"},
    "file":         {"provider": "ollama",     "model": "gemma2:2b"},
    "system":       {"provider": "ollama",     "model": "phi3"},
    "conversation": {"provider": "ollama",     "model": "gemma2:2b"},
    "mcp":          {"provider": "ollama",     "model": "gemma2:2b"},
    "summarizer":   {"provider": "ollama",     "model": "phi3"},
}
```

Supported providers: `ollama`, `openai`, `anthropic`, `google_gemini`, `groq`, `azure_openai`, `openrouter`, `custom_openai`.

## Intent Classification (How the Orchestrator Routes)

The orchestrator runs five stages in order, short-circuiting on the first confident match:

```
① File/folder keywords    "folder", ".csv", "zip the", "send me"…  → FileAgent
② Specialist score ≥ 2   "search"+"latest"+"news" (3 hits)         → ResearchAgent
③ MCP keywords + live     "git", "email", "sql"… AND server up      → MCPAgent
④ Specialist score = 1    single weak match                         → specialist
⑤ LLM classify            sends message + live MCP capabilities     → LLM decides
                           fallback                                  → ConversationAgent
```

File/folder runs before MCP so "send me the downloads folder" never reaches MCPAgent.

## Memory System

Each agent has two memory layers in `memory_store/<agent_name>_memory.json`:

- **Buffer** — last 6 turns verbatim
- **Running summary** — compressed history of older turns (generated by the summarizer LLM)

When the buffer exceeds 10 turns it is compressed into the summary and trimmed. Context sent to the LLM stays small regardless of conversation length.

---

## File Sending on Telegram

When any agent returns `__FILE__:/path/to/file`, the Telegram bot calls `reply_document()` instead of `reply_text()` — the user receives a real downloadable file in chat. This works for generated files, found files, and zipped folders.

The bot must be running on the same machine as the files. Telegram's upload limit is 50 MB.

## MCP Integration

### Architecture

```
MCPAgent (single ReAct loop)
      │
ToolProxyRegistry (started once at boot)
      │  persistent subprocess connections
      │  watchdog thread — pings every 30s, auto-reconnects dead servers
      │
  ┌───┴────────────────────────────────────┐
  │  filesystem  git  fetch  memory  github │  ← running subprocesses
  │  gmail  slack  notion  postgres  sqlite │  ← enabled when keys set
  └────────────────────────────────────────┘
```

MCPAgent gets all tools from all live servers as a flat list and runs a ReAct loop. The LLM decides which tools to call and in what order — no pre-routing inside MCPAgent.

### Enabling MCP servers

Install prerequisites:
```bash
pip install mcp langchain-mcp-adapters
npm install -g npx
pip install uv   # provides uvx
```

Toggle in `.env`:
```env
# On by default (no API key needed):
MCP_FILESYSTEM_ENABLED=true
MCP_GIT_ENABLED=true
MCP_FETCH_ENABLED=true
MCP_MEMORY_ENABLED=true

# Enable by setting credentials:
GITHUB_TOKEN=ghp_...
GMAIL_CLIENT_ID=...
GMAIL_CLIENT_SECRET=...
GMAIL_REFRESH_TOKEN=...
SLACK_BOT_TOKEN=xoxb-...
NOTION_API_KEY=secret_...
POSTGRES_URL=postgresql://...
BRAVE_API_KEY=...
```

### MCP server reference

| Server | Default | Needs |
|---|---|---|
| `filesystem` | ✅ on | nothing |
| `git` | ✅ on | nothing |
| `fetch` | ✅ on | nothing |
| `memory` | ✅ on | nothing |
| `sqlite` | ⬜ off | `MCP_SQLITE_ENABLED=true` |
| `github` | ⬜ off | `GITHUB_TOKEN` |
| `gmail` | ⬜ off | OAuth credentials |
| `slack` | ⬜ off | `SLACK_BOT_TOKEN` |
| `notion` | ⬜ off | `NOTION_API_KEY` |
| `gdrive` | ⬜ off | OAuth credentials |
| `postgres` | ⬜ off | `POSTGRES_URL` |
| `brave_search` | ⬜ off | `BRAVE_API_KEY` |
| `puppeteer` | ⬜ off | `MCP_PUPPETEER_ENABLED=true` |
| `docker` | ⬜ off | `MCP_DOCKER_ENABLED=true` |

### Adding a custom MCP server

Add one entry to `mcp_servers/registry.py`:

```python
"my_server": {
    "transport":    "stdio",
    "command":      "npx",
    "args":         ["-y", "my-mcp-package"],
    "env":          {"MY_API_KEY": os.getenv("MY_API_KEY", "")},
    "description":  "What this server does.",
    "required_env": ["MY_API_KEY"],
    "enabled":      bool(os.getenv("MY_API_KEY")),
},
```

Then add its keywords to `MCP_KEYWORDS` in `agents/orchestrator.py`.

### Reconnecting servers (CLI)

```
/mcp              — show status of all MCP servers
/reconnect        — reconnect all servers
/reconnect github — reconnect one server by name
```

## Telegram Setup

1. Create a bot with [@BotFather](https://t.me/BotFather), get the token
2. Set `TELEGRAM_BOT_TOKEN=...` in `.env`
3. Run `python main.py telegram`

Bot commands: `/start`, `/help`, `/status`, `/mcp`, `/clear`

## WhatsApp Setup

1. Create a Meta Developer App with WhatsApp product
2. Set `WHATSAPP_TOKEN`, `WHATSAPP_PHONE_ID`, `WHATSAPP_VERIFY_TOKEN` in `.env`
3. Run `python main.py whatsapp`
4. Expose port 5000 publicly: `ngrok http 5000`
5. Register webhook URL in Meta dashboard: `https://your-ngrok-url/webhook`
